Modul 9: Web-Analytics und Erfolgsmessung

Jede Interaktion ist ein Ereignis

Der Einstieg bringt das zentrale Prinzip von Google Analytics 4 auf den Punkt: In GA4 ist jede Interaktion ein Ereignis, ein Event. Wer diesen einen Grundgedanken verinnerlicht, versteht die gesamte Logik des Werkzeugs.

Vom Seitenaufruf zum Event

Frühere Werkzeuge dachten in Seitenaufrufen. Die zentrale Frage war: Welche Seite wurde wie oft geöffnet?

GA4 verschiebt diesen Blick. Nicht die Seite ist die kleinste Einheit, sondern die einzelne Handlung:

  • ein Seitenaufruf
  • ein Klick
  • ein Scrollvorgang
  • ein Download

Jede dieser Aktionen wird als Event erfasst.

Quelle: @googleGa4Events2026

Der Perspektivwechsel ist grundlegend. Moderne Websites und Apps bestehen längst nicht mehr nur aus statischen Seiten, die nacheinander laden. Viele Inhalte ändern sich, ohne dass eine neue Seite lädt. Ein eventbasiertes Modell bildet das sauber ab, weil es an die Handlung gebunden ist, nicht an die Seite.

Das Datenmodell besteht aus zwei Teilen

Ein Event nennt die Art der Handlung. Parameter liefern den Kontext dazu.

Event

Was ist geschehen? Zum Beispiel purchase.

Parameter

In welchem Kontext? Bestellwert, Artikel, Seite.

Ein purchase wird erst durch Parameter wie Bestellwert oder Artikel wirklich aussagekräftig.

Quelle: @googleGa4Events2026

Ein Event allein sagt nur, dass etwas geschehen ist. Erst die Parameter machen es auswertbar. Deshalb wird Tracking so entworfen, dass jedes wichtige Event genau die Zusatzangaben mitbringt, die später zur Auswertung gebraucht werden.

Zwei Arten von Events

Einige Events erfasst GA4 automatisch. Andere werden gezielt selbst eingerichtet, weil sie für die eigenen Ziele zentral sind.

Die selbst definierten Ereignisse sind der entscheidende Hebel. Ein automatisch erfasster Seitenaufruf ist nützlich, doch ein bewusst eingerichtetes Event wie ein abgeschicktes Kontaktformular bildet später die Grundlage der wichtigsten Kennzahlen.

Von Events zu Conversions

Nicht jedes Event ist gleich wichtig. Besonders wertvolle Handlungen werden als Conversion markiert.

Aus vielen Events werden wenige Zielhandlungen.

So entsteht aus einer Flut von Ereignissen eine überschaubare Zahl von Conversions, die tatsächlich verfolgt und verbessert werden.

Quelle: @kaushik2010

Dieser Schritt verbindet die technische Erfassung mit den Kennzahlen aus der vorigen Lesson. Es wird bewusst entschieden, welche Events den Rang einer Conversion verdienen. Damit werden sie aus der Masse der Ereignisse herausgehoben und zur eigentlichen Steuerungsgröße.

Analogie: Das Kassenbuch im Einzelhandel

Ein Geschäft zählt nicht nur, wer den Laden betritt.

Es notiert jede Handlung: Regal angesehen, Produkt in die Hand genommen, Frage gestellt, an der Kasse bezahlt.

Genauso protokolliert GA4 nicht bloß den Seiteneintritt, sondern jede relevante Aktion als eigenen Eintrag. Aus diesen Einträgen lässt sich jeder Weg nachvollziehen.

Das Bild des Kassenbuchs macht das eventbasierte Modell greifbar. Statt nur Eintritte zu zählen, wird jede einzelne Handlung festgehalten. Genau diese Granularität erlaubt es später, ganze Pfade der Kundschaft zu rekonstruieren.

Was bleibt

GA4 denkt nicht in Seiten, sondern in Handlungen. Jede Interaktion ist ein Event, beschrieben durch Parameter, und die wertvollsten Events werden zu Conversions. Wer dieses Modell versteht, versteht das ganze Werkzeug.

Als Kernbotschaft bleibt: Das eventbasierte Datenmodell ist der Schlüssel zu GA4. Event, Parameter und Conversion bilden eine durchgängige Kette von der einzelnen Handlung bis zur steuerungsrelevanten Zielgröße.
